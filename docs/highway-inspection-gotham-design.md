# 公路巡检 Codex Demo 设计文档（Gotham 风格重构）

## 1. 设计目标
将 Demo 从“管理后台展示”升级为“沉浸式情报中心”，强调：
- **实体中心化（Object-Centric）**：以路损实体为交互起点。
- **关联图谱（Graph & Linkage）**：展示巡检车、路损点、责任工区之间的闭环关系。
- **战术地理叠加（Tactical Geospatial Overlay）**：地图成为核心交互载体，而非背景。

## 2. 视觉语言
- 基础色：`#0F1012` / `#1A1C1E` 暗色系，减少长时间监控疲劳。
- 功能色：
  - 绿色：正常/安全
  - 琥珀：关注
  - 红色：严重/紧急
  - 青蓝/紫色：选中态与关系链路
- 信息密度：小字号 + 紧凑间距 + 半透明叠层，保持战术仪表风格。

## 3. 信息架构与布局
采用三栏 + 顶部搜索 + 底部时间轴布局：
1. **左栏（对象清单）**：图层开关、路损实体列表、巡检车辆列表。
2. **中栏（地图主视图）**：暗色底图、风险圈层、关系链路、实体标注。
3. **右栏（分析面板）**：关联图谱、Dossier 调查流、任务日志。
4. **顶部（神谕搜索）**：自然语言检索路损事件并定位到地图。
5. **底部（时间旅行轴）**：拖动回放多时段巡检态势演化。

## 4. 关键交互说明
- 点击路损实体（左栏或地图）→ 右侧 Dossier 更新并高亮链路。
- 切换图层（矢量/卫星/热力/链路）→ 地图即时重绘。
- 神谕搜索命中实体 → 地图飞行定位 + 选中实体。
- 时间轴滑动/自动播放 → 车辆位置、路损状态、日志同步变化。

## 5. 演示数据模型
单帧包含：
- `vehicles[]`：巡检车辆（状态、工区、坐标）
- `defects[]`：路损实体（类型、严重度、置信度、建议）
- `logs[]`：当前时段任务事件

通过 `frames[]` 组织多个时段，实现态势“时间旅行”。
