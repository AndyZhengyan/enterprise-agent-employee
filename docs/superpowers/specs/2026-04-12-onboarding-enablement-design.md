# Onboarding + Enablement 设计方案

**日期**: 2026-04-12
**状态**: 已确认

---

## 1. 核心设计原则

**前后端映射原则**：UI 上的每一个控制项，必须能明确对应到 OpenClaw 7 个配置文件的某一部分，不能出现"这个开关改了不知道落在哪个文件"的情况。

```
前端 UI                    后端 Ops API              OpenClaw 文件系统
──────────────────       ──────────────────────    ───────────────────
结构化字段表单      →     组装成 .md 格式      →     ~/.openclaw/agents/
(name, role, tools)        写入对应 .md 文件         <agent-id>/agent/SOUL.md
                                                         IDENTITY.md / AGENTS.md
                                                         TOOLS.md / USER.md
```

## 2. Onboarding 页面 — 整体布局

```
┌──────────────────────────────────────────────────────────┐
│  入职中心                                    + 部署新   │
├────────────────────┬───────────────────────────────────┤
│                    │  [基本信息] [灵魂] [能力] [部署]     │
│  Avatar 列表        │  ──────────────────────────────── │
│  ─────────────     │                                    │
│  ☐ 小白 (运行中)    │  Tab 内容区                        │
│  ☐ 明律 (运行中)    │                                    │
│  ☐ 墨言 (已停止)    │  (随所选 Tab 变化)                  │
│                    │                                    │
│  ─────────────     │                                    │
│  + 部署新 Avatar   │                                    │
└────────────────────┴───────────────────────────────────┘
```

**交互逻辑**：
- **新建**：点击「+ 部署新 Avatar」→ 右侧清空 + 默认展开「基本信息」Tab
- **修改**：点击已有 Avatar → 右侧加载其配置 → 修改 → 保存

## 3. Onboarding Tab 结构 — 字段设计

### Tab 1：基本信息
**映射**: `IDENTITY.md`

| 字段 | 说明 | 读写 |
|------|------|------|
| Blueprint ID | 系统自动生成 | 只读 |
| Avatar 名称 | 如「小白」| 可编辑 |
| 角色 | 如「行政专员」| 可编辑 |
| 部门 | 如「综合管理部」| 可编辑 |

### Tab 2：灵魂
**映射**: `SOUL.md` + `USER.md`

| 字段 | 来源 | 说明 |
|------|------|------|
| 性格描述 | SOUL.md | Avatar 性格/背景 |
| 沟通风格 | SOUL.md | 说话风格 |
| 主要职责 | SOUL.md | 能处理哪些任务 |
| 用户称呼 | USER.md | 服务对象的称呼 |
| 用户偏好 | USER.md | 沟通/排版偏好 |

### Tab 3：能力
**映射**: `AGENTS.md` + `TOOLS.md` + `models.json`

| 字段 | 来源 | 说明 |
|------|------|------|
| 工作流程 | AGENTS.md | 任务处理流程规范 |
| 工具集 | TOOLS.md | 从 Enablement 货架选择允许的工具 |
| 模型 | models.json | Provider + Model 下拉选择 |

**工具集数据来源**：从 Enablement 工具货架读取已注册的工具列表，管理员勾选分配给该 Avatar。

### Tab 4：部署
**汇总预览 + 版本快照**

| 内容 | 说明 |
|------|------|
| 配置汇总 | 各 Tab 填写内容预览 |
| 版本快照 | 保存时自动备份当前文件到 `.backup/` 目录 |
| 部署按钮 | 确认后写入 OpenClaw 文件系统 |

## 4. Enablement 工具货架

**定位**: 工具目录清单，纯展示 + 增删改，无启用/禁用状态（状态在 Onboarding 的 Avatar 能力 Tab 里）

### 初始数据
从 OpenClaw 内置工具列表导入：

| 工具 | 描述 |
|------|------|
| exec / process | 运行 Shell 命令，管理后台进程 |
| code_execution | 沙箱远程 Python 分析 |
| browser | 控制 Chromium 浏览器 |
| web_search | 网络搜索 |
| web_fetch | 抓取网页内容 |
| read / write / edit | 文件读写编辑 |
| apply_patch | 多区块文件补丁 |
| message | 跨渠道发送消息 |
| canvas | Node Canvas 控制 |
| nodes | 发现和定位配对设备 |
| cron | 定时任务管理 |
| gateway | 网关操作 |
| image | 图片分析/生成 |
| music_generate | 音乐生成 |
| video_generate | 视频生成 |
| tts | 文字转语音 |
| subagents | 子 Agent 管理 |
| session_status | 会话状态查询 |

### 界面

```
┌─────────────────────────────────────────────────────────┐
│  赋能中心 · 工具货架                          + 添加工具  │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  web_search              Web 搜索               [编辑] [删除]│
│  file_read               文件读取               [编辑] [删除]│
│  code_exec               代码执行               [编辑] [删除]│
│  git                     Git 操作               [编辑] [删除]│
│  web_fetch               网页内容抓取           [编辑] [删除]│
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 操作
- **查看**: 点击卡片展开详情（名称 + 描述）
- **添加**: 弹窗填入工具名称 + 描述
- **编辑**: 弹窗编辑描述
- **删除**: 确认后删除条目

## 5. 后端组装逻辑

### 写入 OpenClaw 文件
Avatar 配置保存时，后端将前端结构化数据组装成 OpenClaw 文件格式，写入 `~/.openclaw/agents/<agent-id>/agent/` 目录。

### 版本快照
每次部署前，自动备份当前文件：
```bash
~/.openclaw/agents/<agent-id>/agent/SOUL.md
  → ~/.openclaw/agents/<agent-id>/agent/SOUL.md.backup.<timestamp>
```

### 工具集映射
Enablement 货架的工具列表 → Onboarding「能力」Tab 的勾选项 → 写入该 Avatar 的 `TOOLS.md`
