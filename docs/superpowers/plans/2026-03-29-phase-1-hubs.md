# Phase 1 — 能力丰富：ModelHub + ConnectorHub + SkillHub + KnowledgeHub

> **For agentic workers:** Phase 0 验收完成后开始本计划。
> 详细 Task 内容在 Phase 0 验收后根据实际情况细化。

**Goal:** 在 Runtime 核心闭环基础上，接入 ModelHub（多模型路由）、ConnectorHub（连接器管理）、SkillHub（技能中心）、KnowledgeHub（知识中心），让 Runtime 可以调用真实技能和知识。

**Architecture:**
- ModelHub: 多 MaaS 接入 + 按任务类型/成本路由
- ConnectorHub: CLI > MCP > API 连接器优先级体系
- SkillHub: 技能注册 + 生命周期（Draft → Published → Deprecated）
- KnowledgeHub: AgenticRAG，多路召回 + AIGC 知识沉淀

**Tech Stack:** Python 3.11, FastAPI, Redis, Qdrant (Phase 1 末)

---

## Phase 1 里程碑

| 里程碑 | 说明 | 验收条件 |
|--------|------|----------|
| M1 | ModelHub 可用 | Runtime 可通过 ModelHub 调用多个模型 |
| M2 | ConnectorHub 就绪 | CLI/MCP/API 三种连接器可注册和调用 |
| M3 | SkillHub 基础 | 技能可注册，Runtime 可查询可用技能 |
| M4 | KnowledgeHub 基础 | 向量检索可用，Runtime 可查询知识 |
| M5 | 端到端串联 | Gateway → Runtime → 各 Hub 完整链路 |
